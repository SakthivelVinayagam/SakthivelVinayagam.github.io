Stock Return Prediction (Finance)

Predicted short-horizon stock returns using engineered indicators (MA, RSI, volatility) and models including SVM, Random Forest, and Neural Networks. Evaluated with regression and strategy metrics (MSE, Sharpe, drawdown) using walk-forward validation to ensure realistic performance.

  • Case Study (PDF)
  • GitHub

Python · Machine Learning · Time Series Forecasting · Finance · Predictive Modelling

Overview

Built a supervised learning pipeline to forecast short-horizon stock returns from technical indicators. The workflow emphasises robust evaluation (walk-forward, expanding window) and translates model outputs into simple trading rules to report risk-adjusted performance.

  • Target: next-period log return (regression) and up/down (classification variant).
  • Universe: 1–3 liquid equities/ETF (replace with your tickers), daily bars.
  • Horizon: 1–5 trading days ahead (configurable).
  • Stack: Python, pandas, NumPy, scikit-learn, TensorFlow/Keras, matplotlib.

Data & Feature Engineering

  • Inputs: OHLCV data (open, high, low, close, volume) at daily frequency.
  • Indicators: moving averages (SMA/EMA), RSI, MACD, ATR, rolling volatility, momentum, volume spikes, price gaps.
  • Targets: regression – next-period log return; classification – sign of return.
  • Preprocessing: forward-fill, winsorisation of outliers, feature scaling within each training window to avoid leakage.
  • Stationarity aids: log returns and z-scored indicators per window.

Models & Training

I compared linear and non-linear baselines with a simple neural network to balance bias/variance. Hyperparameters were tuned inside each training window to reflect realistic deployment.

  • Baselines: persistence (last return), linear regression / logistic regression.
  • ML: Support Vector Regression / SVM, Random Forest / Gradient Boosting.
  • NN: small MLP (2–3 dense layers with dropout, early stopping).
  • Validation: walk-forward (expanding window) with rolling re-fit; no shuffling.
  • Leakage control: all transforms fit on training window only; targets strictly shifted.

Evaluation & Strategy

Beyond regression loss, I convert predictions into a simple long/flat strategy to assess economic value.

  • Regression: MSE / MAE, R² on walk-forward test slices.
  • Classification variant: accuracy, precision/recall, ROC-AUC.
  • Strategy metrics: cumulative return, Sharpe ratio, max drawdown, hit-rate, turnover (proxy for trading costs).
  • Baselines: buy-and-hold and naive momentum signals for context.

Results

SVM and linear models produced the most stable generalisation, with the MLP competitive but more sensitive to windowing and scaling. On the test period, the model-driven long/flat strategy improved Sharpe over buy-and-hold with lower drawdowns (after modest transaction cost assumptions).

  • Generalisation: consistent MSE gains vs linear baseline; SVM had best hit-rate.
  • Risk: volatility and drawdown reduced vs buy-and-hold.
  • Sensitivity: performance varies with horizon (1–5 day) and indicator set size.

Limitations

  • Results are sensitive to transaction costs, slippage, and borrow constraints.
  • Predictability is regime-dependent; non-stationarity can degrade models quickly.
  • Indicators are price-based; fundamentals/news not included in this version.

Reproducibility & Packaging

  • Config-driven pipeline: data load → features → windowed training → evaluation.
  • Separate fit/transform per window; seeds fixed; report artifacts saved (plots, CSVs).
  • CLI entrypoints for backtests; notebook for EDA.

Next Steps

  • Include fundamentals/news (e.g., earnings surprises, sentiment) for multi-modal signals.
  • Tune horizons and rebalancing cadence; add cost-aware optimisation.
  • Try gradient boosting (XGBoost/LightGBM) and temporal CV with purged/embargoed splits.
  • Calibrate classification thresholds to risk/return targets.
